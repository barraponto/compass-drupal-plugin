A Template Approach

A while ago, Nica Lorba published a blog post with Firework templates attached, named Design for Drupal: A Template Approach. She tried to fix the obvious flaws of almost every Drupal custom theme: unstyled Drupal elements. The templates were meant for designers to quickly notice whether they had thoroughly styled Drupal elements and served as a solid foundation to work on.

This template is wonderful for graphic designers. But I'm a web designer who'd rather design in the browser. I then decided to port the Template Approach to a reusable SCSS stylesheet and noticed that it made porting styles between base themes easier. It might even make porting styles to other CMSs possible, although I'm not pursuing that right now.

How to

First, make sure you have ruby installed on your development environment. WHAT? Yes, this project actually relies on Ruby's implementation of SASS and Compass, but only on your development environment. Please note that you're not required to run ruby code on your server.

Mac users already have Ruby installed. Ubuntu users can just sudo apt-get install ruby. Then let's get started with:

gem install compass-drupal-plugin
compass create -r drupal --using drupal6/zen mytheme --relative-assets

This should create a Zen child theme complete and ready to go. There are three important SCSS files to read and edit in the sass folder:

  • _mytheme-base.scss: settings for sidebar widths, font-size and stacks, fixed footer, sidebar order, etc.
  • _mytheme-skin.scss: the actual template. Instead of looking for selectors like #secondary-menu > li.expanded > ul.menu you will find .-mytheme-secondary-submenu. This abstraction makes it easier to start editing the stylesheets right away, while at the same time makes porting styles between base themes easier.
  • mytheme-skin.scss: maps the base-theme-specific CSS selectors to the skin selectors and applies theme-specific mixins to set sidebar-widths, fixed-footer, sidebar-order, etc. This file should be left untouched unless you want to disable the provided mixins or change smoe mappings to extend styles from a contributed skin file.

Why Drupal 6?

This project started as a SASS port of Zen 6.x-2.x layout stylesheets (because I'm lazy). As this project grew in scope, I decided to finish it's initial roadmap supporting only D6 since porting changes across two code bases would be a burden. Also, once a D7 port is released, it will be very easy to port styles :)
